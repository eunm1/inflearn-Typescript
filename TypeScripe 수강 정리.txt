1. 타입스크립트란?
오픈소스로 공개되어잇는 언어
자바스크립트는 유연한 문법을 사용해서 자유롭고 버그 발생 가능성이 높다
그래서 자바스크립트를 더 안전하게 사용할 수 있도록 타입 관련 기능을 추가한 언어

2. 타입시스템이란?
모든 프로그래밍 언어에는 타입 시스템이 있다
언어의 타입관련된 문법체계
- 정적 타입 시스템(C, Java)
엄격하고 고정적인 시스템, 코드 실행 이전에 변수의 타입을 결정함
빌드시 모든 코드를 확인한다. 코드를 실행하기 전에 정적으로 변수의 타입을 결정
=> 모든 변수에 일일이 타입을 지정해주어야 해서 타이핑 양이 증가하고 유연하지 못하다

- 동적 타입 시스템(Python, JavaScript) 
변수의 타입이 하나로만 고정되지 않는다
빌드시 확인이 안되고 실행 후 에러가 발생할 수 있기때문에 서비스에 치명적이다
코드를 실행하면서 유동적으로 타입을 결정 => 코드의 타입 오류를 미리 검사할 수 없어 예기치 못한 오류가 발생할 수 있다

3. 자바스크립트의 한계점
통적 타입 시스템 사용 - 예기치 못한 오류가 발생할 가능성이 있다

4. 타입스크립트의 극복점
점진적 타입 시스템 : 실행 전 검사를 통한 타입 안정성 확보, 자동으로 변수의 타입을 추론

5. 타입스크립트의 동작원리
대다수의 프로그래밍 언어는 컴퓨터 보다 사람에게 친숙하고 컴파일을 통해 컴퓨터가 이해할 수 있는 기계어로 변경한다
컴파일러는 프로그래밍 언어(JS, Java 등)를 AST(추상 문법트리)로 변환하고 AST를 바이트(byte) 코드(기계어)로 변환하는 과정을 가진다

* 타입스크립트의 컴파일 과정은
TS -> AST -> 타입 검사 -> JS로 변환 -> AST -> byte 코드(기계어) -> 실행
타입검사를 통해 안전한 JS로 변환하여 기존 컴파일러의 동작을 적용한다.
타입 검사에서 오류 발생시 컴파일이 종료되어 오류를 잡아낼 수 있다

*.ts 파일들을 전역 모듈로 판단하기 때문이다.

개별파일을 분리하는 방법
방법 1. 개별 파일에 export나 import를 한번이라도 사용하면 독립모듈로 판단한다
방법 2. 컴파일러 "moduleDetection" : "force" 옵션 추가

{
  "compilerOptions": {
    "target": "ESNext"  //컴파일하여 생성되는 JS의 버전 설정
    ,"skipLibCheck": true 
    // @types 버전이 20버전 이상으로 업데이트되면서 특정 라이브러리에서 타입 검사 오류가 발생하고 있습니다.
    //따라서 tsconfig.json에 compilerOption 내부에 skipLibCheck 옵션을 추가 해 주시기 바랍니다
    , "module": "ESNext" 
    //(option : ESNext or CommonJs) export import 의 형식을 지정한다(기본적으로 ESNext이나 CJS로 설정시 생성된 js 값에 관련 문법이 적용된다)
    , "outDir": "dist" //컴파일시 생성된 js파일들의 경로 지정
    , "strict": true  //TS가 코드를 아주 엄격하게 검사하도록 지정
    , "moduleDetection": "force" 
    // force : 각 ts파일을 독립모듈로 설정한다
    // legacy : 각 ts파일을 전역(글로벌) 모듈로 설정(하나의 파일인것으로) 각 파일에서 동일 변수를 사용할 수 없다
  },

  "include": ["src"] //컴파일 경로 지정, src 하위에 있는 모든 파일을 컴파일 한다

}

- 원시타입 : 하나의 값만 저장하는 타입

- 리터럴 타입
* 값을 타입으로 사용하는 경우
let num: 10 = 10;


{
  "compilerOptions": {
    "strictNullChecks": false
  }
}
true (기본값/권장): null과 undefined를 구체적인 타입으로 간주합니다. 즉, number 변수에는 숫자만 들어갈 수 있습니다.

false: null과 undefined를 모든 타입의 하위 타입으로 간주합니다. 즉, number, string, boolean 등 모든 변수에 null을 할당해도 오류가 나지 않습니다.

더 나은 방법(안전성 확보)
let myNumber: number | null = null;
"이 변수는 숫자일 수도 있고, 비어있을(null) 수도 있다"라고 명시적으로 선언하게 되어 코드가 더 명확해집니다.

chapter 2~7 타입 스크립트 기본

Unknown : 비교적 안전한 any

데이터의 용도나 타입을 확신할 수 없을 때 사용
any 처럼 모든 값을 담을 수 있음
타입정제(type guard)하기 전까지 다른 변수에 할당하거나 메서드를 호출 할 수 없음
Never

값의 끝이 절대 존재하지 않음을 의미
존재하지 않는 불가능한 타입
반환하지 못할때(반환값이 없는게 아닌 불가할때) = 무한루프에 빠져 함수 종료 불가, 항상 예외 throw Error을 던져서 함수가 중간에 멈출때
어떤 값도 담을 수 없음
Void

반환 값이 없을 때 함수 반환 타입에 선언
return 문이 없거나 return 시 아무값도 반환하지 않을때, return undefined일때 사용
Any

타입을 알 수 없을때 임시로 작성
모든 타입을 할당할 수 있다
런타임시 오류 발생가능성이 높다
